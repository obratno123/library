# Документация по проекту «Букинист»

Этот документ составлен по коду из архива проекта и объясняет, **какой файл за что отвечает**, **какие функции есть в проекте** и **что делает каждая ключевая сущность**.

## 1. Общая архитектура

Проект — это Django-приложение онлайн-библиотеки / книжного магазина.

Основные подсистемы:
- **catalog** — каталог книг, поиск, фильтрация, карточки книг, reader.
- **cart_order** — корзина и заказы.
- **users** — регистрация, вход, профиль, редактирование профиля, восстановление пароля, подтверждение почты.
- **review_rating** — отзывы и рейтинг книг.
- **chat** — личные диалоги между пользователями.
- **support_chat** — отдельный чат поддержки.
- **service_entities** — служебные сущности: платежи, история статусов, доступ к файлам книг, сообщения поддержки по заказу.
- **config** — настройки проекта, главный роутинг, стартовые страницы, ASGI-конфигурация.
- **books** — пустое/заглушечное приложение без реализованной бизнес-логики.

Также проект использует:
- **Django Channels + Daphne** для WebSocket-чатов;
- **SMTP** для отправки email;
- **SQLite** как текущую БД;
- **Pillow** для работы с изображениями.

## 2. Корневой уровень проекта

### `manage.py`
Точка входа для команд Django.

#### Функции:
- `main()` — запускает стандартные Django-команды (`runserver`, `migrate`, `test` и т.д.).

### `config/settings.py`
Главный файл настроек проекта.

Что хранится здесь:
- `INSTALLED_APPS` — подключённые приложения (`catalog`, `users`, `chat`, `support_chat`, и др.).
- `TEMPLATES` — путь к HTML-шаблонам (`frontend/templates/html`).
- `DATABASES` — конфигурация SQLite.
- `ASGI_APPLICATION` — точка входа для Channels (`config.asgi.application`).
- `CHANNEL_LAYERS` — слой сообщений для WebSocket (сейчас `InMemoryChannelLayer`).
- SMTP-настройки для email (`EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_HOST_USER`, `EMAIL_HOST_PASSWORD`).
- пути для `MEDIA_ROOT` / `MEDIA_URL` и статики.

### `config/urls.py`
Главный роутер проекта. Здесь подключаются все приложения и маршруты верхнего уровня.

#### Что маршрутизируется:
- `/admin/` → админка Django.
- `/users/` → API/служебные маршруты users-приложения.
- `/catalog/` → каталог книг.
- `/cart/` → корзина и действия с корзиной.
- `/chat/` → чат поддержки.
- `/messages/` → обычные пользовательские диалоги.
- `/reviews/` → отзывы.
- `/` → главная страница.
- `/login/` → HTML-страница входа.
- `/register/` → HTML-страница регистрации.
- `/profile/` → HTML-страница профиля.
- `/profile/edit/` → HTML-страница редактирования профиля.
- `/profile/edit/save/` → сохранение изменений профиля.
- `/profile/verify-email/` → подтверждение email.
- `/profile/verify-email/resend/` → повторная отправка кода подтверждения.

### `config/views.py`
Набор очень простых view для страниц верхнего уровня.

#### Функции:
- `home_page(request)` — рендерит главную страницу `index.html`.
- `login_page(request)` — рендерит HTML-страницу входа `login.html`.
- `register_page(request)` — рендерит HTML-страницу регистрации `register.html`.
- `profile_page(request)` — рендерит HTML-страницу профиля `account.html`.

### `config/asgi.py`
ASGI-конфигурация проекта.

#### Что делает:
- создаёт обычное Django ASGI-приложение для HTTP;
- добавляет WebSocket-маршрутизацию через `ProtocolTypeRouter`;
- подключает websocket-пути из:
  - `support_chat.routing.websocket_urlpatterns`
  - `chat.routing.websocket_urlpatterns`
- использует `AuthMiddlewareStack`, чтобы в WebSocket была доступна аутентификация пользователя.

## 3. Приложение `catalog`

### `catalog/models.py`
Модели, связанные с каталогом книг.

#### `Author`
Автор книги.
- `first_name` — имя.
- `last_name` — фамилия.
- `biography` — биография.
- `birth_date` — дата рождения.
- `__str__()` — отображает автора как `Фамилия Имя`.

#### `Genre`
Жанр книги.
- `name` — название жанра.
- `slug` — slug для URL.
- `__str__()` — возвращает название жанра.

#### `Publisher`
Издательство.
- `name` — название.
- `description` — описание.
- `__str__()` — возвращает название издательства.

#### `Book`
Центральная модель книги.
- `title` — название.
- `slug` — slug для URL.
- `description` — описание.
- `authors` — связь ManyToMany с `Author`.
- `genres` — связь ManyToMany с `Genre`.
- `publish_year` — год издания.
- `isbn` — ISBN.
- `price` — цена.
- `cover_image` — обложка.
- `preview_file` — файл предпросмотра.
- `ebook_file` — полный файл электронной книги.
- `publisher` — ForeignKey на издательство.
- `created_at`, `updated_at` — технические даты.
- `is_active` — активна ли книга в каталоге.
- `__str__()` — возвращает название книги.

#### `Stock`
Остатки книги на складе.
- `book` — OneToOne с `Book`.
- `quantity` — общее количество.
- `reserved_quantity` — зарезервировано.
- `low_stock_threshold` — порог низкого остатка.
- `available()` — считает доступный остаток: `quantity - reserved_quantity`.
- `__str__()` — показывает название книги и доступное количество.

### `catalog/views.py`
View-функции каталога.

#### `book_reader_view(request, slug)`
Открывает страницу чтения электронной книги.

Что делает:
- получает активную книгу по `slug`;
- подгружает авторов;
- формирует строку авторов;
- передаёт в шаблон книгу, список авторов и флаг наличия файла книги (`has_file`).

#### `catalog_home(request)`
Главная страница каталога.

Что делает:
- получает все активные книги;
- подгружает авторов, жанры и издательство;
- читает GET-параметры:
  - `q` — поисковый запрос;
  - `genre` — фильтр по жанру;
  - `author` — фильтр по автору;
  - `publisher` — фильтр по издательству;
  - `sort` — сортировка по цене;
- выполняет поиск по:
  - названию;
  - ISBN;
  - описанию;
  - названию издательства;
  - названию жанра;
  - имени/фамилии автора;
- разбивает поисковую строку на слова и расширяет поиск по каждому слову;
- фильтрует список книг;
- сортирует по цене (`price_asc`, `price_desc`);
- применяет пагинацию по 8 книг;
- рендерит `catalog_home.html`.

#### `book_detail(request, slug)`
Страница одной книги.

Что делает:
- получает книгу по `slug`;
- подгружает авторов, жанры, издательство и отзывы;
- вычисляет наличие на складе;
- подбирает связанные книги по жанрам;
- получает последние отзывы;
- считает средний рейтинг и количество отзывов;
- если пользователь авторизован, ищет его отзыв на эту книгу;
- рендерит `book_detail.html`.

#### `books_by_genre(request, slug)`
Страница книг определённого жанра.
- находит жанр;
- получает книги этого жанра;
- подгружает связанные данные;
- применяет пагинацию;
- рендерит `catalog/books_by_genre.html`.

#### `books_by_author(request, author_id)`
Страница книг конкретного автора.
- находит автора;
- получает его книги;
- пагинирует и рендерит `catalog/books_by_author.html`.

#### `books_by_publisher(request, publisher_id)`
Страница книг издательства.
- находит издательство;
- получает книги этого издательства;
- пагинирует и рендерит `catalog/books_by_publisher.html`.

### `catalog/urls.py`
URL-маршруты каталога.
- `/catalog/` → `catalog_home`
- `/catalog/book/<slug>/` → `book_detail`
- `/catalog/reader/<slug>/` → `book_reader_view`
- `/catalog/genre/<slug>/` → `books_by_genre`
- `/catalog/author/<id>/` → `books_by_author`
- `/catalog/publisher/<id>/` → `books_by_publisher`

### `catalog/management/commands/import_books_csv.py`
Команда импорта книг из CSV и папок с файлами.

#### Класс `Command(BaseCommand)`
Отвечает за импорт данных о книгах.

#### Методы:
- `add_arguments(parser)` — описывает аргументы команды:
  - `--csv`
  - `--epub-dir`
  - `--cover-dir`
  - `--default-quantity`
  - `--default-threshold`
  - `--update-existing`
- `handle(*args, **options)` — основная логика импорта:
  - проверяет существование файлов и директорий;
  - читает CSV;
  - валидирует наличие обязательных колонок;
  - создаёт/обновляет автора, жанр, издательство, книгу и остатки;
  - привязывает файл EPUB и обложку;
  - ведёт счётчики созданных/обновлённых/пропущенных записей.
- `find_epub(epub_dir, number, slug)` — ищет EPUB-файл книги.
- `find_cover(cover_dir, number)` — ищет файл обложки по номеру книги.

## 4. Приложение `cart_order`

### `cart_order/models.py`

#### `Cart`
Корзина пользователя.
- `user` — владелец корзины.
- `created_at`, `updated_at` — даты создания/обновления.
- `__str__()` — строковое представление корзины.

#### `CartItem`
Элемент корзины.
- `cart` — корзина.
- `book` — книга.
- `quantity` — количество.
- `price_at_time` — цена книги на момент добавления.
- `__str__()` — выводит книгу и количество.

#### `Order`
Заказ пользователя.
- `user` — владелец заказа.
- `status` — статус заказа.
- `delivery_method` — способ доставки.
- `payment_method` — способ оплаты.
- `payment_status` — статус оплаты.
- `total_price` — сумма заказа.
- `delivery_address` — адрес доставки.
- `qr_code` — QR-код заказа/оплаты.
- `paid_at` — дата оплаты.
- `created_at`, `updated_at` — даты.
- `__str__()` — строковое представление заказа.

#### `OrderItem`
Позиция заказа.
- `order` — заказ.
- `book` — книга.
- `quantity` — количество.
- `price_at_time` — цена на момент заказа.
- `__str__()` — выводит книгу и количество.

### `cart_order/views.py`

#### `cart_view(request)`
Главная страница корзины.

Что делает:
- получает или создаёт корзину пользователя;
- подгружает элементы корзины и книги;
- считает итоговую сумму;
- рендерит страницу корзины.

#### `add_to_cart(request, book_id)`
Добавляет книгу в корзину.

Что делает:
- получает книгу;
- получает/создаёт корзину;
- если книга уже есть в корзине — увеличивает количество;
- иначе создаёт новую позицию;
- сохраняет цену на момент добавления.

#### `update_cart_item(request, item_id)`
Обновляет количество элемента корзины.

Что делает:
- находит элемент корзины;
- проверяет принадлежность пользователю;
- обновляет количество товара.

#### `remove_cart_item(request, item_id)`
Удаляет позицию из корзины.

Что делает:
- находит элемент корзины;
- проверяет принадлежность пользователю;
- удаляет его.

### `cart_order/urls.py`
- `/cart/` → просмотр корзины.
- `/cart/add/<book_id>/` → добавить книгу.
- `/cart/item/<item_id>/update/` → изменить количество.
- `/cart/item/<item_id>/remove/` → удалить позицию.

## 5. Приложение `users`

### `users/models.py`

#### `Role`
Роль пользователя.
- `name` — название роли.
- используется, например, для роли `support`.
- `__str__()` — возвращает название роли.

#### `Profile`
Расширенный профиль пользователя.
- `user` — связь OneToOne с Django User.
- `full_name` — полное имя.
- `delivery_address` — адрес доставки.
- `city` — город.
- `postal_code` — почтовый индекс.
- `avatar` — аватар пользователя.
- `is_email_verified` — подтверждён ли email.
- `email_verified_at` — когда email был подтверждён.
- `role` — роль пользователя.
- `__str__()` — выводит `full_name`.

#### `PasswordResetCode`
Код восстановления пароля.
- `user` — пользователь.
- `code_hash` — хеш кода.
- `created_at` — дата создания.
- `is_used` — использован ли код.
- `is_expired()` — проверяет, истёк ли код (10 минут).

#### `EmailVerificationCode`
Код подтверждения email.
- `user` — пользователь.
- `email` — почта, для которой создан код.
- `code_hash` — хеш кода.
- `created_at` — дата создания.
- `is_used` — использован ли код.
- `is_expired()` — проверяет срок действия кода.

### `users/views.py`

#### Аутентификация и профиль

##### `register(request)`
Регистрация пользователя по JSON.

Что делает:
- принимает `username`, `password`, `email`;
- валидирует формат email и пароль;
- создаёт пользователя;
- создаёт профиль с `full_name=username`;
- логинит пользователя;
- возвращает JSON с результатом.

##### `user_login(request)`
Вход пользователя по JSON.
- проверяет content-type;
- читает логин и пароль;
- аутентифицирует;
- логинит пользователя;
- возвращает JSON.

##### `user_logout(request)`
Выход пользователя.
- требует авторизацию;
- разлогинивает;
- возвращает JSON.

##### `user_profile(request)`
JSON API профиля текущего пользователя.

Возвращает:
- `user_id`
- `username`
- `email`
- `full_name`
- `city`
- `delivery_address`
- `postal_code`
- `avatar_url`
- `is_email_verified`

##### `edit_profile_page(request)`
HTML-страница редактирования профиля.
- получает/создаёт профиль;
- рендерит `edit_profile.html`.

##### `update_profile(request)`
Сохраняет изменения профиля.

Что делает:
- обновляет `full_name`, `city`, `delivery_address`, `postal_code`;
- при наличии файла обновляет `avatar`;
- если email изменился:
  - сохраняет новый email;
  - сбрасывает флаг подтверждения email;
  - очищает дату подтверждения;
  - помечает старые коды подтверждения как использованные.

#### Восстановление пароля

##### `generate_reset_code()`
Генерирует 6-значный код восстановления.

##### `send_reset_code_email(user, code)`
Отправляет email с кодом восстановления пароля.
- использует шаблоны писем;
- отправляет text + html версии.

##### `send_password_changed_email(user)`
Отправляет уведомление, что пароль был изменён.

##### `password_reset_request_view(request)`
Первый шаг восстановления пароля.
- принимает email;
- сохраняет email в сессии;
- создаёт код восстановления;
- отправляет его на почту;
- редиректит на страницу ввода кода.

##### `password_reset_code_view(request)`
Шаг ввода кода восстановления.
- проверяет код;
- если код верный, сохраняет id пользователя в сессии;
- переводит к форме нового пароля.

##### `password_reset_new_view(request)`
Шаг установки нового пароля.
- валидирует новый пароль;
- сохраняет пароль;
- отправляет email о смене пароля.

##### `password_reset_resend_view(request)`
Повторно отправляет код восстановления пароля.

#### Подтверждение email

##### `generate_verification_code()`
Генерирует 6-значный код подтверждения email.

##### `send_verification_email(user, code)`
Отправляет письмо с кодом подтверждения почты.

##### `verify_email_view(request)`
Страница подтверждения почты.

Что делает:
- если email уже подтверждён, возвращает пользователя в профиль;
- при первом заходе (`GET`) проверяет активный код:
  - если его нет или он истёк, создаёт новый и отправляет письмо;
- при `POST` проверяет введённый код;
- если код верный:
  - помечает код использованным;
  - ставит `is_email_verified=True`;
  - сохраняет дату подтверждения.

##### `resend_verification_email_view(request)`
Повторно отправляет код подтверждения email.
- помечает старые коды как использованные;
- генерирует новый код;
- отправляет письмо;
- возвращает пользователя на страницу подтверждения.

### `users/urls.py`
Маршруты JSON/API и восстановления пароля.
- `/users/register/` → регистрация.
- `/users/login/` → вход.
- `/users/logout/` → выход.
- `/users/profile/` → JSON профиля.
- `/users/profile/edit/` → страница редактирования профиля (дублируется и в верхнем роутинге).
- `/users/profile/edit/save/` → сохранение профиля (дублируется и в верхнем роутинге).
- `/users/password-reset/` → запрос сброса пароля.
- `/users/password-reset/code/` → ввод кода.
- `/users/password-reset/new/` → установка нового пароля.
- `/users/password-reset/resend/` → повторная отправка кода.

## 6. Приложение `review_rating`

### `review_rating/models.py`

#### `Review`
Отзыв пользователя о книге.
- `user` — кто оставил отзыв.
- `book` — на какую книгу отзыв.
- `rating` — оценка.
- `text` — текст отзыва.
- `created_at`, `updated_at` — даты.
- уникальное ограничение: **один пользователь — один отзыв на одну книгу**.
- `stars_range` — range для вывода рейтинга звёздами.

### `review_rating/views.py`

#### `add_or_edit_review(request, slug)`
Добавляет или редактирует отзыв на книгу.

Что делает:
- получает книгу по `slug`;
- принимает `rating` и `text`;
- валидирует рейтинг и текст;
- создаёт новый отзыв или обновляет существующий;
- пишет flash-message об успехе/ошибке;
- возвращает пользователя на страницу книги.

#### `user_reviews(request)`
Показывает все отзывы текущего пользователя.
- получает отзывы пользователя;
- подгружает книгу;
- рендерит `user_reviews.html`.

### `review_rating/urls.py`
- `/reviews/book/<slug>/review/` → добавление/редактирование отзыва.
- `/reviews/my-reviews/` → список отзывов пользователя.

## 7. Приложение `chat`

### `chat/models.py`

#### `Dialog`
Диалог между двумя пользователями.
- `user1`, `user2` — участники диалога.
- `created_at`, `updated_at` — даты.
- `get_other_user(current_user)` — возвращает второго участника.

#### `Message`
Сообщение в пользовательском диалоге.
- `dialog` — диалог.
- `sender` — отправитель.
- `text` — текст.
- `created_at` — дата отправки.
- `is_read` — флаг прочтения.

### `chat/views.py`

#### `dialog_list(request)`
Список пользовательских диалогов.

Что делает:
- получает все диалоги текущего пользователя;
- считает количество сообщений в каждом диалоге;
- показывает только диалоги, где есть сообщения;
- даёт список пользователей, с которыми можно начать новый диалог;
- поддерживает поиск пользователей по username.

#### `start_dialog(request, user_id)`
Начинает диалог с другим пользователем.
- не позволяет создать диалог с самим собой;
- ищет уже существующий диалог между двумя пользователями;
- если его нет — создаёт;
- редиректит на страницу диалога.

#### `dialog_detail(request, dialog_id)`
Страница одного диалога.
- проверяет доступ пользователя к диалогу;
- получает сообщения;
- помечает непрочитанные входящие сообщения как прочитанные;
- рендерит `dialog_detail.html`.

### `chat/consumers.py`

#### `ChatConsumer`
WebSocket consumer для пользовательского чата.

##### Методы:
- `connect()` — проверяет авторизацию, существование диалога и доступ к нему; подключает пользователя к группе WebSocket.
- `disconnect(close_code)` — отключает пользователя от группы.
- `receive(text_data)` — принимает JSON с сообщением, валидирует доступ и создаёт новое сообщение.
- `chat_message(event)` — отправляет клиенту новое сообщение.
- `get_dialog(dialog_id)` — получает диалог из БД.
- `user_has_access(dialog)` — проверяет, является ли текущий пользователь участником диалога.
- `create_message(dialog, sender, text)` — создаёт сообщение и обновляет `updated_at` диалога.

### `chat/routing.py`
WebSocket-маршрут:
- `ws/chat/<dialog_id>/` → `ChatConsumer`

### `chat/urls.py`
- `/messages/` → список диалогов.
- `/messages/start/<user_id>/` → старт диалога.
- `/messages/dialog/<dialog_id>/` → страница диалога.

## 8. Приложение `support_chat`

### `support_chat/models.py`

#### `SupportDialog`
Диалог пользователя с оператором поддержки.
- `user` — клиент.
- `support_user` — оператор.
- `created_at`, `updated_at` — даты.
- уникальность пары `user + support_user`.

#### `SupportMessage`
Сообщение в чате поддержки.
- `dialog` — диалог поддержки.
- `sender` — отправитель.
- `text` — текст.
- `created_at` — дата.
- `is_read` — флаг прочтения.

### `support_chat/views.py`

#### `is_support(user)`
Проверяет, является ли пользователь оператором поддержки.

Логика:
- у пользователя должен быть `profile`;
- в профиле должна быть `role`;
- имя роли должно быть `support`.

#### `choose_support_user()`
Выбирает случайного оператора поддержки.
- получает всех пользователей с ролью `support`;
- если операторов нет — выбрасывает `ValueError`;
- иначе возвращает случайного оператора.

#### `chat_dialog(request)`
Основной пользовательский чат поддержки.

Что делает:
- если текущий пользователь — оператор, редиректит в список диалогов поддержки;
- для обычного пользователя ищет существующий диалог;
- если диалога нет — создаёт его и назначает случайного оператора;
- получает сообщения;
- помечает входящие как прочитанные;
- рендерит `chat_dialog.html`.

#### `support_dialogs_list(request)`
Список диалогов для оператора.
- доступен только пользователю с ролью `support`;
- показывает все его диалоги поддержки.

#### `support_dialog_detail(request, dialog_id)`
Открывает конкретный диалог поддержки для оператора.
- доступен только оператору;
- получает диалог и сообщения;
- помечает входящие как прочитанные;
- рендерит `support_dialog_detail.html`.

### `support_chat/consumers.py`

#### `SupportChatConsumer`
WebSocket consumer для поддержки.

##### Методы:
- `connect()` — проверяет аутентификацию, наличие диалога и права доступа; подключает к группе `support_dialog_<id>`.
- `disconnect(close_code)` — отключает от группы.
- `receive(text_data)` — принимает текст сообщения, создаёт его и рассылает участникам.
- `chat_message(event)` — отправляет сообщение в браузер.
- `get_dialog(dialog_id)` — получает диалог поддержки.
- `user_has_access(dialog)` — проверяет, является ли пользователь участником диалога.
- `create_message(dialog, sender, text)` — создаёт новое сообщение и обновляет `updated_at` диалога.

### `support_chat/routing.py`
WebSocket-маршрут:
- `ws/support-chat/<dialog_id>/` → `SupportChatConsumer`

### `support_chat/urls.py`
- `/chat/` → пользовательский чат поддержки.
- `/chat/support/dialogs/` → список диалогов оператора.
- `/chat/support/dialog/<dialog_id>/` → диалог поддержки для оператора.

## 14. Что важно знать по текущей реализации

1. **Профиль доступен в двух формах**:
   - HTML-страница `/profile/`
   - JSON API `/users/profile/`

2. **Маршруты профиля частично дублируются**:
   - верхнеуровневые HTML-маршруты идут через `config/urls.py`;
   - часть маршрутов продублирована в `users/urls.py`.

3. **Подтверждение email запускается из профиля**, а не автоматически при регистрации.

4. **Чаты разделены на два типа**:
   - `chat` — обычные личные диалоги;
   - `support_chat` — диалог с оператором поддержки.

5. **service_entities** хранит служебные сущности, но не содержит отдельного пользовательского интерфейса.
